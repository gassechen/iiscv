# Documentación de dump-source-code.lisp

## Descripción General

Proporciona funciones para exportar/dump del código fuente almacenado en los grafos de IISCV. Permite reconstruir archivos Lisp desde el historial de commits.

## Ubicación

=src/dump-source-code.lisp=

## Paquete

=iiscv=

## Dependencias

- =main.lisp= (grafos)
- =utility-fn.lisp= (búsqueda)

## Estado

Completamente documentado

---

## Funciones Principales

### dump-source-code (&optional output-dir)

**Propósito:** Exporta todo el código fuente de commits humanos a un archivo lineal.

**Características:**
- Extrae solo las versiones más recientes de cada símbolo
- Evita duplicación de código
- Genera un solo archivo =dump.lisp=

**Proceso:**
1. Itera sobre =*human-history-graph*= en orden topológico
2. Para cada hito, procesa sus atomic-uuids
3. Para cada commit atómico:
   - Extrae la forma fuente
   - Si el símbolo no ha sido procesado, lo agrega a =final-forms=
4. Escribe todas las formas al archivo de salida
5. Añade timestamp deámetros:**
- generación

**Par =output-dir=: Directorio de salida (default: =output_src/=)

**Retorna:** Número de formas escritas

**Ejemplo de salida:**
#+begin_src lisp
;; Generated by IISCV dump-source-code
;; Timestamp: 3979401818

(DEFUN FOO (X) (+ X 1))
(DEFUN BAR (Y) (* Y 2))
...
#+end_src

---

### dump-source-code-by-commit-type (&optional output-dir)

**Propósito:** Exporta el código fuente agrupado por tipo de commit.

**Tipos de archivo generados:**

| Tipo | Archivo |
|------|---------|
| function | functions.lisp |
| variable | variables.lisp |
| type | types.lisp |
| slot-change | slot-changes.lisp |
| dependency | dependencies.lisp |

**Características:**
- Un archivo por tipo de definición
- Un archivo =main.lisp= que carga todos los archivos
- Evita duplicación (solo última versión)

**Proceso:**
1. Itera sobre todos los hitos
2. Para cada commit atómico:
   - Determina el tipo usando =get-commit-type=
   - Agrupa por tipo en hash table
3. Genera =main.lisp= con declaraciones =load=
4. Genera archivo individual por tipo
5. Escribe todas las formas

**Parámetros:**
- =output-dir=: Directorio de salida (default: =output_src/=)

**Retorna:** NIL

---

## Estructura de Archivos Generados

### main.lisp (loader)
#+begin_src lisp
;; Generated by IISCV dump-source-code-by-commit-type
;; Timestamp: 3979314987

(load "functions.lisp")
(load "variables.lisp")
(load "types.lisp")
...
#+end_src

### functions.lisp
#+begin_src lisp
;; Function definitions
;; Generated by IISCV

(DEFUN FOO (X) ...)
(DEFUN BAR (Y) ...)
...
#+end_src

## Comparación de Funciones

| Característica | dump-source-code | dump-source-code-by-commit-type |
|----------------|------------------|--------------------------------|
| Archivos | 1 | Múltiples |
| Agrupación | No | Sí (por tipo) |
| Uso recomendado | Dump simple | Reconstrucción modular |

---

## Algoritmo de Deduplicación

Ambas funciones usan el mismo algoritmo:

1. Hash table =processed-symbols= para tracking
2. Solo procesa símbolos que NO están en el hash
3. Al procesar, marca el símbolo como =t=
4. Ignora versiones anteriores del mismo símbolo

**Ventaja:** Solo se incluye la versión más reciente de cada definición.

## Integración con Otros Módulos

- =register-commit-type.lisp=: Proporciona =get-commit-type=
- =main.lisp=: Proporciona =*human-history-graph*= y =*atomic-history-graph*=
- =make-image.lisp=: Usa dump para reconstrucción

## Ejemplo de Uso

#+begin_src lisp
;; Dump lineal simple
(dump-source-code)

;; Dump agrupado por tipo
(dump-source-code-by-commit-type)

;; Dump a directorio específico
(dump-source-code-by-commit-type #p"mi-proyecto/src/")
#+end_src

---

## Notas

- Los dumps son útiles para generar código legible desde el grafo
- La reconstrucción de imagen (=rebuild-image-from-human-history=) usa las formas directamente, no los dumps
- Los dumps son complementarios, no el mecanismo principal de reconstrucción
- El timestamp permite identificar cuándo fue generado el dump
